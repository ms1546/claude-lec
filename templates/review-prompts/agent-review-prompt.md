# Agent レビュープロンプト

## 包括的コードレビュー用プロンプト

```
現在のコードベースを包括的にレビューしてください。

レビュー対象：
- /src ディレクトリ配下のすべてのコンポーネント
- 設定ファイル（package.json, tsconfig.json等）
- テストファイル

レビュー観点：
1. アーキテクチャの一貫性
2. セキュリティ脆弱性
3. パフォーマンスボトルネック
4. 依存関係の健全性
5. テストカバレッジ

以下の形式で報告してください：

## 優先度: 高
- 問題点
- 影響範囲
- 改善提案

## 優先度: 中
- 問題点
- 改善提案

## 優先度: 低
- 改善提案
```

## セキュリティ監査用プロンプト

```
セキュリティの観点から以下を重点的に確認してください：

1. 認証・認可の実装
   - トークンの管理方法
   - セッション管理
   - 権限チェックの実装

2. データ検証
   - 入力値検証
   - SQLインジェクション対策
   - XSS対策

3. 外部APIとの通信
   - HTTPS使用の確認
   - APIキーの管理
   - CORS設定

4. 依存関係
   - 既知の脆弱性を持つパッケージ
   - 最新のセキュリティパッチ

問題を発見した場合は、CVSSスコアと共に報告してください。
```

## パフォーマンス分析用プロンプト

```
アプリケーションのパフォーマンスを分析してください：

1. レンダリング最適化
   - 不要な再レンダリング
   - メモ化の機会
   - 遅延ローディングの可能性

2. バンドルサイズ
   - 大きなdependencies
   - コード分割の機会
   - Tree shakingの最適化

3. データフェッチング
   - N+1問題
   - キャッシング戦略
   - 並列処理の機会

4. 画像・アセット
   - 最適化されていない画像
   - 適切なフォーマットの使用

改善による推定パフォーマンス向上率も報告してください。
```

## リファクタリング提案用プロンプト

```
コードの保守性向上のためのリファクタリングを提案してください：

1. コードの重複
   - 共通化できるロジック
   - 再利用可能なコンポーネント

2. 複雑性の削減
   - 循環的複雑度が高い関数
   - 深いネストの解消

3. 型安全性の向上
   - any型の使用箇所
   - 型定義の改善機会

4. テスタビリティ
   - テストしづらい実装
   - モックしやすい設計への改善

各提案について、実装の難易度（簡単/中/難）と
推定作業時間を含めてください。
```
